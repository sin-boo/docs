Neutral Art's AI features require proper configuration to work. This guide covers all the settings you can adjust to customize your AI assistant experience.

## AI Provider Settings

### Google Studio / Gemini

<ResponseField name="api_key" type="string" required>
Your Google Studio API key. Get one from [Google AI Studio](https://makersuite.google.com/app/apikey).

<Note>This is currently the only fully supported provider in Neutral Art.</Note>
</ResponseField>

<ResponseField name="model" type="string" default="gemini-pro">
Which Gemini model to use for feedback. Available options:
- `gemini-pro` — Standard model (recommended)
- `gemini-pro-vision` — Vision-capable model
</ResponseField>

<ResponseField name="temperature" type="number" default="0.7">
Controls randomness in AI responses. Lower values (0.1-0.3) for consistent, focused feedback. Higher values (0.7-1.0) for more creative, varied responses.
</ResponseField>

## Interface Settings

<ResponseField name="panel_position" type="string" default="top">
Where the AI assistant panel appears in Neutral Art:
- `top` — At the top of the workspace (current default)
- `right` — As a right-side docker panel
- `floating` — As a floating window
</ResponseField>

<ResponseField name="auto_hide" type="boolean" default="false">
Automatically hide the AI panel when not in use to save workspace space.
</ResponseField>

<ResponseField name="hide_api_key" type="boolean" default="true">
Hide your API key in the interface when screen sharing or recording.

<Warning>Always enable this for privacy when sharing your screen!</Warning>
</ResponseField>

## Feedback Preferences

<ResponseField name="default_feedback_type" type="string" default="general">
What type of feedback the AI provides by default:
- `general` — Overall composition and technique
- `technical` — Detailed technical analysis
- `constructive` — Focus on improvement areas
- `encouraging` — Motivational feedback
</ResponseField>

<ResponseField name="include_examples" type="boolean" default="true">
Whether AI suggestions include specific examples and demonstrations.
</ResponseField>

<ResponseField name="feedback_depth" type="string" default="balanced">
How detailed the AI feedback should be:
- `brief` — Quick, high-level feedback
- `balanced` — Standard detailed feedback (recommended)
- `comprehensive` — Very detailed analysis
</ResponseField>

## Canvas & Layer Settings

<ResponseField name="default_send_mode" type="string" default="layer">
What gets sent to AI by default when requesting feedback:
- `canvas` — Entire canvas
- `layer` — Currently selected layer
- `selection` — Selected area only
</ResponseField>

<ResponseField name="max_image_size" type="string" default="1024px">
Maximum dimension for images sent to AI. Larger images provide better analysis but use more API quota.
- `512px` — Fast, low detail
- `1024px` — Balanced (recommended)
- `2048px` — High detail, slower
</ResponseField>

<ResponseField name="auto_resize" type="boolean" default="true">
Automatically resize images to fit within the maximum size limit.
</ResponseField>

## Privacy & Security

<ResponseField name="local_storage_only" type="boolean" default="true">
Store API keys and settings only locally on your device. Never send to external servers.

<Check>Neutral Art never transmits your API keys to our servers — all AI communication happens directly between your computer and your chosen AI provider.</Check>
</ResponseField>

<ResponseField name="clear_on_exit" type="boolean" default="false">
Clear API keys from memory when Neutral Art closes. More secure but requires re-entering keys each session.
</ResponseField>

<ResponseField name="anonymize_data" type="boolean" default="true">
Remove any personal information from feedback requests when possible.
</ResponseField>

## Performance Settings

<ResponseField name="cache_responses" type="boolean" default="true">
Cache AI responses for similar requests to reduce API usage and improve speed.
</ResponseField>

<ResponseField name="max_concurrent_requests" type="number" default="1">
Maximum number of simultaneous AI requests. Increase for faster processing but higher API usage.
</ResponseField>

<ResponseField name="timeout_seconds" type="number" default="30">
How long to wait for AI responses before timing out (in seconds).
</ResponseField>

## Troubleshooting

<AccordionGroup>
  <Accordion title="AI not responding">
    - Check your internet connection
    - Verify your API key is valid and has quota remaining
    - Try switching to a different AI model
    - Check the Neutral Art debug logs
  </Accordion>

  <Accordion title="Canvas sending not working">
    - Ensure you have an active layer selected
    - Check that your image isn't too large (reduce max_image_size if needed)
    - Try sending a smaller selection instead of the full canvas
  </Accordion>

  <Accordion title="Poor quality feedback">
    - Try a different AI model or temperature setting
    - Send higher resolution images (increase max_image_size)
    - Be more specific in your questions to the AI
  </Accordion>

  <Accordion title="API quota exceeded">
    - Check your Google Studio usage dashboard
    - Consider upgrading your API plan
    - Reduce request frequency or image sizes
    - Enable response caching to reduce duplicate requests
  </Accordion>
</AccordionGroup>

## Advanced Configuration

<ResponseField name="custom_endpoints" type="object">
Override default API endpoints for advanced users. Only modify if you know what you're doing.

<Expandable title="Custom Endpoints">
  <ResponseField name="google_studio_base" type="string">
  Custom Google Studio API base URL (rarely needed)
  </ResponseField>
  <ResponseField name="timeout_override" type="number">
  Custom timeout in milliseconds
  </ResponseField>
</Expandable>
</ResponseField>

<ResponseField name="debug_mode" type="boolean" default="false">
Enable detailed logging for troubleshooting. Creates debug files in your Neutral Art data folder.

<Warning>Debug mode may impact performance and create large log files. Disable when not needed.</Warning>
</ResponseField>

<Tip>
Most users won't need to change the advanced settings. The default configuration works well for most workflows. Only modify these if you're experiencing specific issues or have advanced requirements.
</Tip>

<ResponseField name="api" type="API">
  Configuration for API settings. Learn more about API pages at [API Components](/api-playground/demo).
  <Expandable title="API">
    <ResponseField name="baseUrl" type="string">
      The base url for all API endpoints. If `baseUrl` is an array, it will enable for multiple base url
      options that the user can toggle.
    </ResponseField>

    <ResponseField name="auth" type="Auth">
      <Expandable title="Auth">
        <ResponseField name="method" type='"bearer" | "basic" | "key"'>
          The authentication strategy used for all API endpoints.
        </ResponseField>
        <ResponseField name="name" type="string">
        The name of the authentication parameter used in the API playground.

        If method is `basic`, the format should be `[usernameName]:[passwordName]`
        </ResponseField>
        <ResponseField name="inputPrefix" type="string">
        The default value that's designed to be a prefix for the authentication input field.

        E.g. If an `inputPrefix` of `AuthKey` would inherit the default input result of the authentication field as `AuthKey`.
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="playground" type="Playground">
      Configurations for the API playground

      <Expandable title="Playground">
        <ResponseField name="mode" default="show" type='"show" | "simple" | "hide"'>
          Whether the playground is showing, hidden, or only displaying the endpoint with no added user interactivity `simple`

          Learn more at the [playground guides](/api-playground/demo)
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="maintainOrder" type="boolean">
      Enabling this flag ensures that key ordering in OpenAPI pages matches the key ordering defined in the OpenAPI file.

      <Warning>This behavior will soon be enabled by default, at which point this field will be deprecated.</Warning>
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="openapi" type="string | string[]">
  A string or an array of strings of URL(s) or relative path(s) pointing to your
  OpenAPI file.
  
  Examples:
  <CodeGroup>
    ```json Absolute
    "openapi": "https://example.com/openapi.json"
    ```
    ```json Relative
    "openapi": "/openapi.json"
    ```
    ```json Multiple
    "openapi": ["https://example.com/openapi1.json", "/openapi2.json", "/openapi3.json"]
    ```
  </CodeGroup>

</ResponseField>

<ResponseField name="footerSocials" type="FooterSocials">
  An object of social media accounts where the key:property pair represents the social media platform and the account url.
  
  Example: 
  ```json
  {
    "x": "https://x.com/mintlify",
    "website": "https://mintlify.com"
  }
  ```
  <Expandable title="FooterSocials">
    <ResponseField name="[key]" type="string">
    One of the following values `website`, `facebook`, `x`, `discord`, `slack`, `github`, `linkedin`, `instagram`, `hacker-news`
    
    Example: `x`
    </ResponseField>
    <ResponseField name="property" type="string">
    The URL to the social platform.
    
    Example: `https://x.com/mintlify`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="feedback" type="Feedback">
  Configurations to enable feedback buttons

  <Expandable title="Feedback">
    <ResponseField name="suggestEdit" type="boolean" default="false">
    Enables a button to allow users to suggest edits via pull requests
    </ResponseField>
    <ResponseField name="raiseIssue" type="boolean" default="false">
    Enables a button to allow users to raise an issue about the documentation
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="modeToggle" type="ModeToggle">
  Customize the dark mode toggle.
  <Expandable title="ModeToggle">
    <ResponseField name="default" type={'"light" or "dark"'}>
      Set if you always want to show light or dark mode for new users. When not
      set, we default to the same mode as the user's operating system.
    </ResponseField>
    <ResponseField name="isHidden" type="boolean" default="false">
      Set to true to hide the dark/light mode toggle. You can combine `isHidden` with `default` to force your docs to only use light or dark mode. For example:
      
      <CodeGroup>
      ```json Only Dark Mode
      "modeToggle": {
        "default": "dark",
        "isHidden": true
      }
      ```

      ```json Only Light Mode
      "modeToggle": {
        "default": "light",
        "isHidden": true
      }
      ```
      </CodeGroup>

    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="backgroundImage" type="string">
  A background image to be displayed behind every page. See example with
  [Infisical](https://infisical.com/docs) and [FRPC](https://frpc.io).
</ResponseField>
